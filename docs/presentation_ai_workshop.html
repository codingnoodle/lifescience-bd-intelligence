Vibe Coding

Prompt 1: Architecture in one shot

One comprehensive prompt scaffolds the entire backend

What the prompt tells Claude Code

◆FastAPI + LangGraph + Claude Sonnet stack ◆4-node sequential pipeline with clear ownership ◆Fixed vocabularies (9 signals, 5 verdicts) ◆Three-scenario valuation formula ◆State schema with ~25 fields
Example prompt to Claude CodeI want to build a BD intelligence tool. When a user types a drug name, the system should: 1. Parse the asset and its indications 2. Research the science (MOA, trial data, PTRS) 3. Find comparators and size the market 4. Identify likely buyers and price the deal Use 4 agents in a pipeline. Each agent should only own its piece — no overlap. Use lookup tables for PTRS math, not the LLM.

What it produces (~10 files)

📄state.py — shared TypedDict schema 📄graph.py — compiled LangGraph pipeline 📄4 agent files with prompts + Tavily search 📄PTRS lookup + buyer context tables 📄FastAPI with SSE streaming endpoint

Key: Be specific about ownership and formulas. Vague prompts produce vague agents.

Sonnet 4.5

Sonnet 4.5
How we instruct each agentThe science agent should act like a senior biotech scientist — profile the asset, score each indication, but stay in its lane. It should NOT touch market sizing or buyers. Those belong to the next agents downstream.
03

Design Principle 3/5

Let code do math, let AI do reasoning

🤖

AI identifies signals

"This is an orphan drug" (+7%)

"Has breakthrough designation" (+5%)

"Strong Phase 2 data" (+10%)

💻

Python calculates

13% base + 7% + 5% + 10% = 35%
Science agent promptFor each indication, look for de-risking signals like orphan drug status, breakthrough designation, strong Phase 2 data, etc. Just identify the signals — don't calculate the probability. The system has lookup tables that handle the math from there.
Market agent promptSearch for the current standard of care and leading competitors. Show why you picked that comparator. Let code derive the peak sales number — e.g. "$3.5B = Scemblix $2.8B x 1.25".
04

Design Principle 4/5

Show your work

Users don't trust a black box

Every number traces back to a specific assumption:

🔍

COMPARATOR

"Scemblix — highest efficacy in same population"

🎲

PROBABILITY

"Base 13%, adjusted to 32% from 5 signals"

💰

DEAL PRICE

"$7B = $3.5B peak sales x 2.0x multiple"

The Dual-View Pattern

Show two numbers side by side:

Risk-adjusted

$2.2B

Investment decision

If-succeed

$7.0B

Negotiation price

Users can edit ANY assumption and recalculate instantly.

Synthesizer promptIdentify which big pharma buyers have a strategic gap this asset fills. Use patent cliff timelines and flush-capital signals to score urgency. Show every assumption — the user should be able to edit any number and recalculate instantly.
05

Design Principle 5/5

Ground in real data, not vibes

❌ The anchoring problem

Telling the AI: "Deals are typically $2-5B"

Result: ALWAYS outputs $2-5B regardless of the asset.

Bad vibe coding promptTell the agent that deals in oncology are usually $2-5B. Price the asset accordingly.

✅ Comparable reasoning

Give it raw comparable transactions and let it reason about similarity.

The AI assesses: "How similar is this asset to each comp?"

Vibe coding promptGive the synthesizer a table of 27 real 2025-2026 M&A deals. It should rank the top 3 most similar by mechanism, stage, and competitive dynamics, then derive the deal multiple from those comps.

What the prompt describes

◆Welcome screen with hero + use-case cards ◆Filter sidebar with chip selectors ◆SSE streaming chat with progress ◆Dual-view valuation waterfall ◆Editable assumptions panel ◆API response shape to consume
Example from the promptBuild a React + Tailwind frontend that consumes the /analyze/stream SSE endpoint. Show a dual-view waterfall chart: - Purple bars = risk-adjusted values - Amber bars = if-succeed values Include an AssumptionsPanel where users can edit PTRS, verdict, peak sales and hit "Recalculate" to call /recalculate.

Why describe the UI to AI?

You're not writing CSS — you're describing what the user sees and does.

📑 Describe layout structure
🔌 Describe each component's purpose
📦 Describe the data shape from API
👆 Describe interactions (click, edit, submit)

Vibe Coding

Prompt 3: Validate against real deals

Test your system against known prices — without hardcoding answers

4 blind test cases

1 TERN-701 — displacement (Merck $6.7B)
2 Centessa ORX750 — category creation ($7.8B)
3 Akero/Novo — emerging TA, MASH ($5.2B)
4 Chimerix/Jazz — rare disease ($935M)
Example test criteriaTERN-701: system must discover Scemblix as comparator, identify CML displacement thesis, and produce if-succeed value within 30% of the actual $6.7B Merck deal price.

What validation catches

🔍Wrong comparator → search query too broad
⚖️Wrong verdict → rubric thresholds off
📈Price too low → deal multiple needs tuning
📉Price too high → capture rate too aggressive
🎯

Target: Strategic if-succeed within 30% of actual deal price
