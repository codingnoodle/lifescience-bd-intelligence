Roadmap

Limitations and what comes next

Current Limitations

  • Training data has a knowledge cutoff -- very recent trial readouts may lag by weeks
  • Paywalled journal articles are not directly accessible; relies on abstracts and public data
  • No live ClinicalTrials.gov API integration yet -- trial status is point-in-time
  • Valuations are indicative ranges, not investment recommendations

On the Roadmap

  • Portfolio mode -- analyze 10+ assets in a single run with comparative rankings
  • Deal tracker integration -- auto-update when new deals are announced
  • Automated alerts -- notify when an asset in your watchlist has a material event
  • Conference scanner -- ingest ASCO, ASH, AACR abstracts in real time
Enterprise Path

From POC to enterprise deployment

Phase 1: POC (Today)

  • 3-agent sequential pipeline
  • 27 real M&A deal benchmarks
  • Dual-view valuation with editable assumptions
  • Validated within 20-30% of real deal prices

Phase 2: Pilot

  • SSO / RBAC -- enterprise auth, team-level access
  • Portfolio mode -- batch-analyze 20+ assets, comparative ranking
  • Audit trail -- log every assumption, override, and recalculation
  • Internal data connectors -- CRM, deal tracker, proprietary forecasts

Phase 3: Scale

  • Live data feeds -- ClinicalTrials.gov API, SEC filings, conference abstracts
  • Automated alerts -- watchlist monitoring, material event triggers
  • Multi-tenant -- separate deal rooms per team/fund
  • Custom models -- fine-tuned on your firm's historical deal data

The architecture is designed to scale -- specialized agents mean you can upgrade any single component (search, scoring, pricing) without rebuilding the system.
